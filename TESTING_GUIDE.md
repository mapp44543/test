## 3. ADVANCED ТЕСТЫ (с DevTools)

### Тест 3.1: Проверить event listeners accumulation

**Шаги**:
1. Откройте браузер DevTools (F12)
2. Перейдите на вкладку Console
3. Скопируйте и выполните этот код:

```javascript
// Подсчёт количества слушателей
const getEventListeners = (element, event) => {
  const listeners = getEventListeners(element);
  return (listeners[event] || []).length;
};

// Запустите несколько раз при zoom/pan
console.log('mousemove listeners:', getEventListeners(window, 'mousemove'));
console.log('mouseup listeners:', getEventListeners(window, 'mouseup'));
console.log('wheel listeners:', getEventListeners(document.querySelector('.map-container'), 'wheel'));
```

**Ожидание**:
- ✅ Количество слушателей = 1 для каждого события
- ✅ Количество НЕ растёт при zoom/pan (было бы 100+ в старом коде)

**Если не работает**:
- Старые listener'ы всё ещё добавляются
- Проверьте что dependency array в useEffect пустой `[]`

### Тест 3.2: Performance Profiling

**Шаги**:
1. Откройте DevTools → Performance вкладка
2. Нажмите кнопку Record (чёрный кружок)
3. Быстро выполните zoom + pan (5-10 секунд)
4. Нажмите Stop
5. Дождитесь анализа

**Что смотреть**:
- Chart должен показать низкую высоту (мало времени на JavaScript)
- Линия FPS должна быть высокой (60 FPS идеально, минимум 50)
- JavaScript execution time < 3-5ms per frame

**Если не работает**:
- Если FPS низкий, есть ещё проблемы с производительностью
- Посмотрите на "Flamechart" - какой код занимает больше всего времени

---

### Тест 3.3: React DevTools Profiler

**Шаги**:
1. Откройте браузер DevTools
2. Перейдите на вкладку "Profiler" (если установлено React DevTools)
3. Нажмите кнопку Record (красный кружок)
4. Выполните zoom несколько раз
5. Нажмите Stop

**Что смотреть**:
- `OfficeMap` component - должен рендериться < 20ms
- `VirtualizedMarkerLayer` - должен рендериться < 50ms

**Если не работает**:
- Если много re-render'ов, есть ещё проблемы с зависимостями
- Посмотрите на "Renders" список - какие компоненты пересчитываются

## 6. ОБНАРУЖЕННЫЕ ПРОБЛЕМЫ И РЕШЕНИЯ

### Проблема: "Зависание при быстром zoom"

**Возможные причины**:
1. RAF loop не оптимизирована
2. Canvas не используется для большого количества маркеров
3. Есть ещё скрытые listener'ы

**Решения**:
1. Проверьте что `wheelRafIdRef` правильно отменяется
2. Убедитесь что Canvas используется для 150+ маркеров
3. Запустите тест 3.1 для проверки listener'ов

### Проблема: "Дёргание при панорамировании"

**Возможные причины**:
1. `isPanningRef` не обновляется правильно
2. `rafIdRef` для mouse move имеет конфликт

**Решения**:
1. Проверьте что `handleMouseDown` устанавливает `isPanningRef.current = true`
2. Проверьте что `handleMouseUpRef` устанавливает `isPanningRef.current = false`

### Проблема: "Карта не следует за мышью"

**Возможные причины**:
1. `handleMouseMoveRef.current` не инициализирована
2. `panPositionRef` не обновляется

**Решения**:
1. Убедитесь что первый useEffect (инициализация refs) выполнен
2. Проверьте что `panPositionRef.current = { x: newX, y: newY }` выполняется
